# 🩺 家庭照護紀錄

給多位家人輪流照顧病人使用的手機網頁應用程式。全家人打開同一個網址，就能**即時看到彼此記錄的資料**。

## 功能

- **快速記錄（滾輪輸入）**：所有數字都用滾輪選擇，不必打字。生命徵象一個表單同時記錄體溫/血壓/血氧/心率（可取消沒量的項目），也有「只量體溫」捷徑；異常數值自動標紅並警示（體溫 ≥38°C 或 <35°C、收縮壓 ≥140 或 <90、血氧 <94%、心率 >100 或 <60）
- **發燒 SOP**：記錄到體溫 ≥38°C 時，會要求確認「退燒藥吃了嗎」「護士知道了嗎」，並寫入紀錄備註
- **血糖紀錄**：滾輪輸入 mg/dL，可標記量測時機（空腹/飯前/飯後/睡前）；≥200 或 <70 自動警示，含趨勢圖與 CSV 匯出
- **加入主畫面**：右上角 📲 按鈕一鍵安裝成 App（iPhone 顯示 Safari 操作步驟）；長按 App 圖示有「記錄血糖」「記錄小便」等捷徑直接跳到記錄畫面
- **出入量追蹤**：喝水/飲品、小便（分尿壺/廁所）、大便（分排便椅/廁所，可勾「有小便」）
- **小便倒數計時**：每記錄一次小便就重新倒數（間隔可在設定中調整）；大便有勾「有小便」也會重置倒數
- **趨勢圖表**：各項數值折線圖，可切換 今日 / 7 天 / 30 天；血壓圖同時顯示收縮壓與舒張壓
- **排班表**：週曆視圖，每位家人有代表色；首頁顯示「目前值班者」與「下一班」
- **交班日誌**：時間軸留言，交接病人狀況
- **匯出 CSV**：一鍵匯出生命徵象與日誌，回診時給醫師參考（Excel 可直接開啟，中文不亂碼）
- 行動裝置優先、大字體大按鈕、全繁體中文

## 開始使用（一次性設定，約 15 分鐘）

### 第 1 步：建立 Supabase 免費資料庫（存放共享資料）

1. 到 [supabase.com](https://supabase.com) 用 Email 或 GitHub 註冊免費帳號
2. 按「New project」建立專案（名稱隨意，例如 `takecare`；資料庫密碼自訂並記下；區域選 Northeast Asia (Tokyo) 較快）
3. 等專案建立完成後，點左側選單的 **SQL Editor** → 貼上本專案 [`setup.sql`](./setup.sql) 的完整內容 → 按 **Run**
4. 點左側 **Project Settings → API Keys**，複製兩個值：
   - **Project URL**（形如 `https://xxxx.supabase.co`）
   - **anon public** 金鑰（一長串 `eyJ...`）

> 免費方案的容量對家庭使用綽綽有餘，不需要信用卡。

### 第 2 步：啟用 GitHub Pages（取得手機可開的網址）

1. 打開這個 GitHub repo 的 **Settings → Pages**
2. 「Source」選 **Deploy from a branch**，Branch 選 `main`（資料夾 `/ (root)`）→ Save
3. 約 1–2 分鐘後，網址就會生效：`https://<你的帳號>.github.io/takecare/`

### 第 3 步：第一位家人完成設定

1. 用手機打開上面的網址
2. 選「**雲端同步模式**」，貼上 Project URL 與 anon 金鑰
3. 自訂一組**家庭代碼**（例如 `lin-family-2026`，全家人共用這組代碼）
4. 填病人稱呼與您自己的稱呼 → 開始使用

### 第 4 步：邀請其他家人

1. 在 App 右上角「⚙️ 設定」→「邀請家人加入」→ **分享給家人**（LINE、簡訊皆可）
2. 家人在手機上點開邀請連結，輸入自己的稱呼，就自動完成設定、看到同一份資料
3. 建議大家把網頁「**加入主畫面**」，之後像 App 一樣一鍵開啟

## 資料同步方式

- 資料存放在你們自己的 Supabase 資料庫，App 每 20 秒自動更新一次，切回頁面時也會立即更新
- 手機暫時沒網路時仍可瀏覽最近一次同步的資料；恢復連線後自動更新
- 也可以先選「本機模式」單機使用（資料只存手機），之後在設定中改為雲端同步，既有資料會自動上傳合併

## 隱私與安全須知

- 資料以「家庭代碼」區隔，只有知道代碼與網址的人能看到你們的資料——**請勿公開分享邀請連結**
- 這是為家庭私人使用設計的簡易共享機制，請避免在備註中存放身分證字號等高敏感資料
- 所有資料都在你們自己的 Supabase 專案裡，可隨時到 Supabase 後台查看、備份或刪除

## 資料庫升級（既有使用者，可選）

App 內建相容機制：即使資料庫還沒有血糖欄位，血糖也能正常記錄與顯示（暫存於備註欄位，讀取時自動解析）。
若想讓血糖存進正式欄位，可到 Supabase 專案的 **SQL Editor** 執行一行（或重新執行整份 `setup.sql`）：

```sql
alter table public.vitals add column if not exists glucose integer;
```

執行後新紀錄自動改用正式欄位，先前的紀錄也仍可正常判讀。
